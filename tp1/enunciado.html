O tesouro do Barba Negra - será possível reparti-lo equitativamente?

  • Tarefa A – Solução baseada em abordagem sôfrega
  • Tarefa B – Solução baseada em análise de combinações
  • Tarefa C – Solução baseada em programação dinâmica

 

Descrição do Problema

Dois amigos descobriram na praia um baú com um grande tesouro enterrado pelo famoso pirata Barba Negra – joias, pedras preciosas e moedas em ouro e platina de imenso valor.

Os dois amigos resolvem dividir o tesouro da forma mais justa possível, repartindo entre si os vários objetos encontrados no baú. No entanto, os objetos são em tão grande número que eles têm dificuldade em garantir que a distribuição que fizeram é a mais equitativa.

Para os ajudar, construa uma aplicação que:

·      recebe como input uma lista com o valor de N objetos (o valor do baú).

·      apresenta como output a menor diferença que é possível encontrar entre o valor de dois subconjuntos que contenham, entre si, a totalidade desses objetos.

Formato do Input:

A primeira linha do input indica o número total de objetos do conjunto.

O valor de cada objeto é indicado nas linhas seguintes (um objeto por linha), sendo garantido que os objetos estão sempre ordenados por ordem decrescente do seu valor.

 

Exemplo A (conjunto com 4 objetos, com os valores {10, 6, 5, 2})

4

10

6

5

2

 

Exemplo B (conjunto com 5 objetos, com os valores {5, 5, 4, 3, 3})

5

5

5

4

3

3

 

Tarefa A :: Solução baseada numa abordagem sôfrega

Na Tarefa A deverá usar uma abordagem sôfrega baseada na seguinte heurística:

·      Os objetos do tesouro são dispostos por ordem decrescente.

·      O primeiro amigo fica com o objeto mais valioso.

·      O segundo amigo fica com o 2º objeto mais valioso.

·      O objeto remanescente de maior valor fica para o amigo com menor valor acumulado.

·      Este último passo é sucessivamente repetido, até repartir todo o tesouro.

·      O output corresponde à diferença entre a soma do valor de cada subconjunto.

Como poderá constatar no output correspondente ao exemplo B, esta solução é simples e aparentemente eficiente, ainda que nem sempre produza a melhor solução possível.

 

Output correspondente ao Exemplo A (amigo  #1:{10, 2}, amigo #2: {6, 5})

1[\n]

 

Output correspondente ao Exemplo B (amigo  #1:{5, 4}, amigo #2: {5, 3, 3})

2[\n]

 

Tarefa B :: Solução baseada numa abordagem de “força-bruta”

Na Tarefa B deverá usar uma abordagem baseada em “força-bruta”, que estime todas  as formas possíveis de repartir os objetos, de modo a determinar qual a menor diferença que pode ser encontrada na repartição do tesouro.

Esta abordagem deverá usar recursividade para explorar as várias formas possíveis de repartir os objetos, tendo em conta que:

·      O 1º objeto pode ficar para o amigo #1 ou para o amigo #2.

·      Para cada uma das duas opções relativamente ao 1º objeto, o 2º objeto pode igualmente ficar para o amigo #1 ou para o amigo #2, dando um total de 4 soluções possíveis para distribuir os primeiros 2 objetos.

·      E assim sucessivamente... até chegar ao último objeto.

Dado que esta abordagem é exaustiva, existe a garantia de encontrar efetivamente a menor diferença possível entre os 2 conjuntos, como pode ser verificado no exemplo B.

 

Output correspondente ao Exemplo A (amigo  #1:{10, 2}, amigo #2: {6, 5})

1[\n]

 

Output correspondente ao Exemplo B (amigo  #1:{5, 5}, amigo #2: {4, 3, 3})

0[\n]

 

São possíveis algumas optimizações nesta abordagem (por exemplo parar quando se encontra uma solução com diferença ZERO, mesmo que ainda não se tenham explorado todas as combinações possíveis), que poderá OPCIONALMENTE introduzir na sua solução e mencionar/analisar no relatório final.

 

Tarefa C :: Solução baseada em programação dinâmica

Na Tarefa C deverá usar uma abordagem baseada em programação dinâmica, tirando partido das seguintes observações:

·      Se o valor total do tesouro for SOMA (por exemplo 460), existe uma repartição equitativa se e só se existir um subconjunto com valor SOMA/2 (por exemplo 230).

·      Caso assim não seja, a melhor repartição tem uma diferença de 2 caso exista um subconjunto cujo valor seja SOMA/2-1 (por exemplo 229).

·      Caso assim não seja, a melhor repartição tem uma diferença de 4 caso exista um subconjunto cujo valor seja SOMA/2-2 (por exemplo 228).

Este raciocínio mantém-se para tesouros de valor ímpar, com as devidas adaptações.

Para operacionalizar este raciocínio, pode usar um array auxiliar, de tipo Boolean e com tamanho SOMA+1. Cada elemento i deste array terá valor TRUE (caso exista um subconjunto cujo valor seja i) ou FALSE (caso não exista nenhum subconjunto nestas condições). Para construir esse array auxiliar de forma eficiente, considere que:

·      Todos os elementos do array começam por ter o valor FALSE.

·      Cada elemento do tesouro em análise permite que existam subconjuntos com os seguintes valores asseridos a TRUE:

o   O próprio valor do elemento em análise (posição array[valor_do_elemento]).

o   Para cada um dos subconjuntos já encontrados anteriormente, a soma do valor desse subconjunto com o valor do elemento em questão.

O output desta Tarefa deverá ser idêntico ao output da Tarefa B.

 

Output correspondente ao Exemplo A (soma total = 23, melhor subconjunto = 11):

1[\n]

 

Output correspondente ao Exemplo B (soma total = 20, melhor subconjunto = 10):

0[\n]

 

Tal como na Tarefa B, são possíveis algumas optimizações nesta abordagem (por exemplo: será que vale a pena analisar subconjuntos de soma > SOMA/2? Porquê?), que poderá OPCIONALMENTE introduzir na sua solução e mencionar/analisar no relatório final.
